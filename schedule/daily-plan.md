  - 上次练习: `reviews/2026-06-05/`（回顾场景法练习中的扣分点）
- **核心知识点**:
  - **基本流** = 正常业务流程，一切顺利的"快乐路径"（happy path）
  - **备选流** = 基本流中每个步骤可能出现的分支/异常情况

  - 四个场景类型：正常的、备选的、异常的、假定推测的

### 主题 2: Postman 进阶 — Collection / Environment / Tests 脚本 🆕

- **所属分类**: 四、接口与 API 测试 [P0]
- **优先级**: P0 ★ （接口测试核心工具，直接用于日常工作）
- **预计时长**: 40 分钟
- **为何今天学这个**: 上次 Postman 入门练习（6/4）得分 63/100，主要问题是 Body 与 Header 概念混淆。现在纠正基础后，进入 Postman 的三大核心进阶功能：Collection（用例组织）、Environment（环境切换）、Tests（自动化断言）。这是从"手工发请求"到"半自动化接口测试"的关键一步。
- **学习资源**:
  - 课件: `md/test/软件测试课件/第4章节 用例篇.md`（Postman/接口测试相关部分）
  - 课件: `md/test/软件测试课件/接口自动化测试精品课课件.md`（第 1-2 章）
  - Postman 官方文档（如需）：https://learning.postman.com/docs/
- **核心知识点**:
  - **Collection** = 接口用例集合，按功能模块组织，可导出/分享/用 Newman 命令行运行
  - **Environment** = 环境变量管理（如 `{{base_url}}`、`{{token}}`），开发/测试/生产环境一键切换
  - **Pre-request Script** = 请求发送前执行的脚本（如自动生成签名、获取时间戳）
  - **Tests 脚本** = 请求发送后执行的断言脚本，常用：
    - `pm.test("用例名", () => { pm.expect(...).to... })`
    - `pm.response.to.have.status(200)` — 断言状态码
    - `pm.expect(pm.response.json()).to.have.property("id")` — 断言返回结构
    - `pm.expect(jsonData.name).to.eql("预期值")` — 断言字段值
  - **Variables** 作用域：Global > Collection > Environment > Data > Local

  - [ ] 在 Postman 中创建一个 Collection，添加至少 4 个请求（15 分钟）
  - [ ] 配置 Environment 变量并使用 Tests 脚本写断言（15 分钟）
- **实践练习**:

  > 1. **创建 Collection**：名为"JSONPlaceholder 测试"，包含以下 4 个请求：
  >    - `GET 帖子列表` — `GET /posts`
  >    - `GET 单个帖子` — `GET /posts/1`
  >    - `POST 创建帖子` — `POST /posts`，Body: `{"title": "test", "body": "test body", "userId": 1}`
  >    - `GET 按用户筛选` — `GET /posts?userId=1`
  > 2. **配置 Environment**：创建环境 `JSONPlaceholder-Dev`，设置变量 `base_url` = `https://jsonplaceholder.typicode.com`，将所有请求的 URL 改为 `{{base_url}}/posts` 形式。
  > 3. **编写 Tests 脚本**：为以上 4 个请求分别编写断言：

  >    - POST 创建帖子：断言状态码 201、返回的 `id` 字段存在
  >    - GET 按用户筛选：断言状态码 200、返回数组中每一项的 `userId` 都等于 1
