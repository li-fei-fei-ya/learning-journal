## 二、推荐学习路线（实习生前 3 个月）

### 路线总览

```
第 1 个月：打地基
测试基础理论 → 流程化测试 → 等价类+边界值 → Bug 提交规范
                                              ↘
                                              SQL 基础查询

第 2 个月：上工具
Postman 接口测试 → 接口用例设计 → Python 基础语法
                                       ↘
                                   requests 脚本

第 3 个月：深入领域
Pytest 接口自动化入门 → Agent 基础概念 → Agent 功能测试入门
```

### 本周推荐：3 个优先学习主题

#### 推荐 1: 测试基础理论 + 流程化测试 (P0 ★ — 最高优先级)

- **为什么先学**：这是所有测试工作的基石。公司日常就是需求评审→写用例→执行→提 Bug→写报告，必须对齐这个流程。
- **前置依赖**: 无
- **预计时长**: 5-7 天（每天 1-1.5h）
- **学习资源**:
  - `md/test/软件测试课件/第1章节 认识测试.md` — 测试定义与岗位认知
  - `md/test/软件测试课件/第2章节 概念篇.md` — 软件生命周期、开发模型、V/W 模型
  - `md/test/软件测试课件/第3章节 BUG篇.md` — BUG 概念、级别、生命周期、评审
  - `md/test/软件测试课件/第4章节 用例篇.md` — 测试用例编写、等价类、边界值
  - `md/test/软件测试课件/第5章节 测试分类.md` — 单元/集成/系统/验收、冒烟/回归
- **核心知识点** (从 topics.md 筛选):
  - 软件测试定义、测试七项原则、测试金字塔
  - V 模型 / W 模型（理解测试在开发各阶段的位置）
  - BUG 生命周期 (New→Open→Fixed→Closed→Reopen)
  - 测试用例要素：测试环境、操作步骤、预期结果
  - 等价类划分法（有效/无效）、边界值分析法
  - 冒烟测试 vs 回归测试
- **实践练习**:
  > 选一个你正在用的 App 功能（如登录页），写 8-10 个测试用例，覆盖正常流程、边界值（空输入、超长输入、特殊字符）、异常场景（网络断开、超时）。按"前置条件→操作步骤→预期结果"格式编写。

#### 推荐 2: SQL 基础查询 (P0 — 第二优先)

- **为什么先学**：接口测试需要查数据库验证结果，Bug 定位需要查数据，测试数据需要自己造。SQL 是每天都会用到的技能。
- **前置依赖**: 无
- **预计时长**: 5-7 天（每天 1h）
- **学习资源**:
  - `md/mysql/MySQL课件-2024/1. 数据库基础.md` — 数据库概念、SQL 分类
  - `md/mysql/MySQL课件-2024/2. 库的操作.md` — CREATE/ALTER/DROP DATABASE
  - `md/mysql/MySQL课件-2024/4. 表的操作.md` — CREATE TABLE、数据类型
  - `md/mysql/MySQL课件-2024/5. 增删改查操作.md` — **重点：SELECT/WHERE/JOIN/GROUP BY**
- **核心知识点**:
  - SELECT / WHERE / LIKE / BETWEEN / IN
  - ORDER BY / LIMIT / OFFSET 分页
  - GROUP BY / HAVING / 聚合函数 (COUNT/SUM/AVG/MAX/MIN)
  - INNER JOIN / LEFT JOIN / RIGHT JOIN
  - INSERT / UPDATE / DELETE
- **实践练习**:
  > 搭建本地 MySQL，创建一张"用户表"和一张"订单表"，插入 20+ 条测试数据。完成 5 个查询练习：① 查找某个日期之后的订单 ② 统计每个用户的订单数 ③ 查询订单金额 Top 10 ④ 关联查询用户名+订单详情 ⑤ 用 GROUP BY 统计每日订单总额。

#### 推荐 3: Postman 接口测试 (P0 — 第三优先)

- **为什么先学**：接口测试是实习生最快能独立产出价值的技能。学会 Postman 后可以直接参与公司的接口测试任务。
- **前置依赖**: 测试基础理论（建议先完成推荐 1 的测试设计技术部分）
- **预计时长**: 5-7 天（每天 1h）
- **学习资源**:
  - `md/test/软件测试课件/第4章节 用例篇.md` — 接口测试部分（Postman/GET/POST/请求URL/请求头/请求体）
  - `md/test/软件测试课件/接口自动化测试精品课课件.md` — 接口概念、用例设计、requests、Pytest
- **核心知识点**:
  - HTTP 协议基础：GET/POST/PUT/DELETE、状态码 (2xx/4xx/5xx)
  - Postman 基本操作：Collection、Environment、Tests 脚本
  - 接口测试用例设计：参数校验、组合场景、异常覆盖
  - JSON 格式、请求头/请求体
- **实践练习**:
  > 找一个公开 API（如 JSONPlaceholder 或公司内部测试 API），用 Postman 设计 10 个接口测试用例：
  > - 3 个通过性验证（正常参数，验证 200 和返回结构）
  > - 3 个参数校验（缺参数、错类型、越界值，验证 400）
  > - 2 个鉴权验证（无 Token/错 Token，验证 401/403）
  > - 2 个业务场景链（创建→查询→更新→删除 完整链路）
  > 将用例导出为 Postman Collection。
