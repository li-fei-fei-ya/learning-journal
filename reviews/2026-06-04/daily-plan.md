### 主题 1: 边界值分析法 — 重新巩固 🔄

- **所属分类**: 三、测试设计技术 [P0]
- **优先级**: P0 ★ （昨日复习暴露薄弱，今优先补强）
- **预计时长**: 40 分钟
- **为何今天学这个**: 昨日复习（6/3）边界值上点/离点概念完全反了，掌握度从 3 降至 2，是当前最紧急的薄弱点。正确理解边界值是所有用例设计的基础。
- **学习资源**:
  - 课件: `md/test/软件测试课件/第4章节 用例篇.md`（**重读** 3.2.3 边界值分析法 部分，重点关注上点/离点/内点的定义和取值规则）
  - 昨日复习记录: `reviews/2026-06-01/2026-06-01.md`（回顾 Q3 的错误分析）
- **核心知识点**:
  - **上点** = 边界上的点（边界本身），如范围 [18, 60]，上点是 18 和 60
  - **离点** = 离边界最近但不在边界上的点，如 17（下离点）、61（上离点）
  - **内点** = 边界范围内的任意一点，如 30
  - 边界值取值口诀：**"上点全取、离点各取最近两个、内点取一个"**
  - 边界值分析 = 等价类划分的补充，不是替代——先划等价类，再对每个等价类的边界取值

- **实践练习**:
  > **边界值分析专项练习 — 三道不同场景的题目**，强化上点/离点/内点的正确使用：

### 主题 2: Postman 接口测试入门 🆕

- **所属分类**: 四、接口与 API 测试 [P0]
- **优先级**: P0 ★ （实习生最快独立产出价值的技能）
- **预计时长**: 35 分钟
- **为何今天学这个**: 测试设计技术（等价类+边界值+判定表+场景法）已轮过一遍，前置依赖已满足。接口测试是自然延续，也是缺口分析中的 P0 优先项。此前 6/1 计划预告本周进入接口测试，现在接续推进。
- **学习资源**:
  - 课件: `md/test/软件测试课件/第4章节 用例篇.md`（阅读 Postman/接口测试相关部分）
  - 课件: `md/test/软件测试课件/接口自动化测试精品课课件.md`（阅读 第1-2章：接口测试概念 + HTTP 基础）
- **核心知识点**:
  - HTTP 请求方法：GET（获取数据）、POST（创建数据）、PUT（更新数据）、DELETE（删除数据）
  - HTTP 状态码：2xx（成功）、4xx（客户端错误，如 400 参数错误/401 未认证/404 未找到）、5xx（服务端错误）
  - URL 结构：协议 + 域名 + 端口 + 路径 + 查询参数（`?key=value`）
  - 请求头 (Headers) 与请求体 (Body) 的区别：Header 放元信息（如 Content-Type: application/json），Body 放业务数据
  - Postman 基本操作：创建 Request → 选择 Method → 填 URL → 加 Headers → 填 Body → Send → 看 Response

- **实践练习**:

  >
  > 1. **发送一个 GET 请求**：获取帖子列表 `GET /posts`，观察 Response 的 Status Code、Body（JSON 格式）、Headers
  > 2. **发送一个带参数的 GET 请求**：查询单个帖子 `GET /posts/1` 和按用户筛选 `GET /posts?userId=1`，对比两次返回的区别
  > 3. **发送一个 POST 请求**：创建新帖子 `POST /posts`，Body 为 JSON 格式 `{"title": "测试标题", "body": "测试内容", "userId": 1}`，观察返回的状态码和 Body 中是否包含 `id: 101`
  > 4. **截图记录**：截取每个请求的 Response 面板（含 Status、Body），粘贴到输出文件中
  > 5. **总结**：用自己的话回答——参数放 URL 和放 Body 有什么区别？
