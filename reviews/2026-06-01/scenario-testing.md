## 修正参考

### 任务 1: 流程图（基本流 + 备选流，标注分岔与回归）

```
基本流: 浏览商品 → 加入购物车 → 进入结算页 → 选择地址 → 选择支付方式 → 确认订单 → 支付成功

备选流 1 (购物车为空):
  分岔点: 进入结算页时检测购物车为空
  流程: 系统提示"购物车为空，请先添加商品" → 返回商品浏览页
  回归: 重新加入商品后回到基本流"进入结算页"

备选流 2 (地址未填写):
  分岔点: 选择地址时地址列表为空
  流程: 系统提示"请先添加收货地址" → 跳转地址管理页 → 填写并保存地址
  回归: 返回结算页继续"选择支付方式"

备选流 3 (支付失败):
  分岔点: 确认订单后输入支付密码
  流程: 密码错误 → 系统提示"支付失败，请重试" → 可重试3次
  回归: 重新输入密码 → 支付成功；或超过3次 → 订单保留待支付

备选流 4 (库存不足):
  分岔点: 加入购物车时检测库存
  流程: 系统提示"该商品库存不足" → 商品无法加入购物车
  回归: 浏览其他商品或等待补货
```

### 任务 3: 方法对比

场景法：从用户视角设计端到端的业务流程，通过基本流（正常路径）+备选流（异常/分支路径）模拟真实使用场景。解决的是**业务流程完整性**的问题——用户能不能从头到尾走通一个业务。

等价类/边界值：从数据视角划分输入域，关注单个/少量输入控件的取值覆盖。解决的是**输入数据的充分性**问题——各种数据值系统是否正确处理。

两者互补：等价类/边界值保证"每个输入框都测到位"，场景法保证"用户的操作路径都走得通"。
